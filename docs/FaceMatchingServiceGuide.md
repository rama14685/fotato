## Introduction

The Face Matching Service provides the core face recognition algorithm for comparing customer face embeddings with photo face embeddings. This service implements cosine similarity calculations for 128-dimensional face embedding vectors and provides efficient batch processing capabilities.

### Key Features

- **Cosine Similarity Calculation**: Accurate mathematical comparison of face embeddings
- **Batch Processing**: Efficient processing of multiple photos against a single customer embedding
- **Multiple Faces Per Photo**: Automatically handles photos with multiple detected faces
- **Configurable Thresholds**: Flexible similarity threshold configuration
- **Memory Efficient**: Chunked processing for large albums (>5000 photos)
- **Thread Safe**: Stateless design for concurrent usage
- **Comprehensive Error Handling**: Detailed exceptions with context information

### Prerequisites

- PHP 8.1 or higher
- Laravel 11.x
- Face embeddings generated by face-api.js (128-dimensional float arrays)
- Laravel Crypt for encryption/decryption of stored embeddings

## Basic Usage

### Example 1: Simple Face Matching with Single Customer and Multiple Photos

This example demonstrates the most common use case: matching a customer's face against photos in an album.

```php
<?php

use App\Services\FaceMatching\FaceMatchingService;
use App\Services\FaceMatching\CosineSimilarityCalculator;
use App\Services\FaceMatching\DTOs\PhotoEmbeddingData;

// Initialize the service
$calculator = new CosineSimilarityCalculator();
$service = new FaceMatchingService($calculator);

// Customer embedding (128-dimensional array from face-api.js)
$customerEmbedding = [
    0.123456, -0.789012, 0.345678, /* ... 125 more values ... */
];

// Prepare photo embeddings
$photoEmbeddings = [
    new PhotoEmbeddingData(
        photoId: 1001,
        embeddings: [
            [0.125, -0.790, 0.348, /* ... 125 more */], // Face 1
        ]
    ),
    new PhotoEmbeddingData(
        photoId: 1002,
        embeddings: [
            [0.987, -0.654, 0.321, /* ... 125 more */], // Face 1
            [0.555, -0.333, 0.777, /* ... 125 more */], // Face 2
        ]
    ),
    new PhotoEmbeddingData(
        photoId: 1003,
        embeddings: [
            [0.120, -0.785, 0.340, /* ... 125 more */], // Face 1
        ]
    ),
];

// Match faces using default threshold (0.6)
$results = $service->matchFaces($customerEmbedding, $photoEmbeddings);

// Process results
foreach ($results as $result) {
    echo "Photo ID: {$result->photoId}\n";
    echo "Similarity: " . round($result->similarityScore, 4) . "\n";
    echo "Matches: " . ($result->matchesThreshold ? 'Yes' : 'No') . "\n";
    echo "---\n";
}

// Output example:
// Photo ID: 1001
// Similarity: 0.8745
// Matches: Yes
// ---
// Photo ID: 1003
// Similarity: 0.7234
// Matches: Yes
// ---
// Photo ID: 1002
// Similarity: 0.4521
// Matches: No
// ---
```

## Error Handling Patterns

### Example 8: Basic Error Handling

```php
<?php

use App\Services\FaceMatching\Exceptions\InvalidEmbeddingException;
use App\Services\FaceMatching\Exceptions\InvalidThresholdException;

try {
    $results = $service->matchFaces(
        $customerEmbedding,
        $photoEmbeddings,
        $threshold
    );
    
    // Process results
    foreach ($results as $result) {
        // Handle each result
    }
    
} catch (InvalidEmbeddingException $e) {
    // Handle invalid embedding errors
    Log::error('Invalid embedding in face matching', [
        'error' => $e->getMessage(),
        'context' => $e->getContext(),
    ]);
    
    // Return empty results or show error to user
    return [];
    
} catch (InvalidThresholdException $e) {
    // Handle invalid threshold errors
    Log::error('Invalid threshold value', [
        'error' => $e->getMessage(),
        'context' => $e->getContext(),
    ]);
    
    // Use default threshold instead
    $results = $service->matchFaces($customerEmbedding, $photoEmbeddings);
    
} catch (\Throwable $e) {
    // Handle unexpected errors
    Log::error('Unexpected error in face matching', [
        'error' => $e->getMessage(),
    ]);
    
    return [];
}
```

## Integration with Laravel Encryption Layer

The Face Matching Service works with **decrypted** embeddings. Encryption and decryption are handled by the caller using Laravel's encryption services.

### Example 13: Decrypting Customer Embedding

```php
<?php

use Illuminate\Support\Facades\Crypt;
use App\Models\User;

// Retrieve user with encrypted face embedding
$user = User::find($userId);

// Decrypt the customer embedding
// The UserFaceEmbedding model uses encrypted casting
$customerEmbedding = $user->userFaceEmbedding->embedding_data;

// The embedding_data accessor automatically decrypts the data
// It returns a 128-dimensional float array ready for face matching

// Now use with face matching service
$results = $service->matchFaces($customerEmbedding, $photoEmbeddings);
```

### Example 14: Decrypting Photo Embeddings

```php
<?php

use App\Models\Photo;
use App\Models\FaceEmbedding;
use App\Services\FaceMatching\DTOs\PhotoEmbeddingData;

// Retrieve photos from an album
$photos = Photo::where('album_id', $albumId)
    ->with('faceEmbeddings')
    ->get();

// Prepare photo embeddings with decryption
$photoEmbeddings = [];

foreach ($photos as $photo) {
    // Get all face embeddings for this photo
    $embeddings = [];
    
    foreach ($photo->faceEmbeddings as $faceEmbedding) {
        // Decrypt embedding data
        // The FaceEmbedding model uses encrypted casting
        $embeddings[] = $faceEmbedding->embedding_data;
    }
    
    // Skip photos with no face embeddings
    if (empty($embeddings)) {
        continue;
    }
    
    // Create PhotoEmbeddingData DTO
    $photoEmbeddings[] = new PhotoEmbeddingData(
        photoId: $photo->id,
        embeddings: $embeddings
    );
}

// Now use with face matching service
$results = $service->matchFaces($customerEmbedding, $photoEmbeddings);
```

### Example 15: Complete Workflow with Encryption

```php
<?php

use App\Models\User;
use App\Models\Album;
use App\Services\FaceMatching\FaceMatchingService;
use App\Services\FaceMatching\CosineSimilarityCalculator;
use App\Services\FaceMatching\DTOs\PhotoEmbeddingData;

class FaceSearchService
{
    public function __construct(
        private FaceMatchingService $faceMatchingService
    ) {}
    
    /**
     * Search for photos containing a customer's face in an album
     */
    public function searchPhotosForCustomer(int $userId, int $albumId, ?float $threshold = null): array
    {
        // 1. Retrieve and decrypt customer embedding
        $user = User::with('userFaceEmbedding')->findOrFail($userId);
        
        if (!$user->userFaceEmbedding) {
            throw new \Exception('Customer has no face embedding registered');
        }
        
        // Automatically decrypted by model accessor
        $customerEmbedding = $user->userFaceEmbedding->embedding_data;
        
        // 2. Retrieve and decrypt photo embeddings
        $album = Album::with(['photos.faceEmbeddings'])->findOrFail($albumId);
        
        $photoEmbeddings = [];
        
        foreach ($album->photos as $photo) {
            if ($photo->faceEmbeddings->isEmpty()) {
                continue;
            }
            
            // Collect all face embeddings for this photo (automatically decrypted)
            $embeddings = $photo->faceEmbeddings
                ->map(fn($fe) => $fe->embedding_data)
                ->toArray();
            
            $photoEmbeddings[] = new PhotoEmbeddingData(
                photoId: $photo->id,
                embeddings: $embeddings
            );
        }
        
        // 3. Perform face matching
        $results = $this->faceMatchingService->matchFaces(
            $customerEmbedding,
            $photoEmbeddings,
            $threshold
        );
        
        // 4. Filter to matched photos only
        $matchedResults = array_filter($results, fn($r) => $r->matchesThreshold);
        
        // 5. Return photo IDs and similarity scores
        return array_map(fn($result) => [
            'photo_id' => $result->photoId,
            'similarity' => round($result->similarityScore, 4),
        ], $matchedResults);
    }
}
```

### Important Security Notes

1. **Never log raw embedding values** - The service automatically excludes embedding values from logs for privacy protection
2. **Decrypt before passing to service** - The service expects plain numeric arrays, not encrypted strings
3. **Store embeddings encrypted** - Always use Laravel's encrypted casting for database storage
4. **Validate decrypted data** - The service validates dimensions and numeric values automatically

## Performance Optimization

### Best Practices

1. **Reuse Service Instance**: Use dependency injection and singleton binding for better performance
2. **Pre-filter Photos**: Only include photos with face embeddings in the batch
3. **Use Chunked Processing**: For albums >5000 photos, use `matchFacesChunked()`
4. **Batch Multiple Customers**: Process multiple customers against the same album in sequence
5. **Monitor Performance**: Log processing times and photo counts for optimization

### Example 20: Optimized Album Search

```php
<?php

class OptimizedFaceSearchService
{
    public function __construct(
        private FaceMatchingService $faceMatchingService
    ) {}
    
    public function searchAlbum(int $userId, int $albumId): array
    {
        // 1. Retrieve customer embedding
        $user = User::with('userFaceEmbedding')->findOrFail($userId);
        $customerEmbedding = $user->userFaceEmbedding->embedding_data;
        
        // 2. Efficiently retrieve only photos with face embeddings
        $photos = Photo::where('album_id', $albumId)
            ->whereHas('faceEmbeddings') // Only photos with faces
            ->with('faceEmbeddings')
            ->get();
        
        // 3. Prepare photo embeddings
        $photoEmbeddings = $photos->map(function ($photo) {
            return new PhotoEmbeddingData(
                photoId: $photo->id,
                embeddings: $photo->faceEmbeddings
                    ->map(fn($fe) => $fe->embedding_data)
                    ->toArray()
            );
        })->toArray();
        
        // 4. Choose processing method based on size
        $photoCount = count($photoEmbeddings);
        
        if ($photoCount > 5000) {
            $results = $this->faceMatchingService->matchFacesChunked(
                $customerEmbedding,
                $photoEmbeddings
            );
        } else {
            $results = $this->faceMatchingService->matchFaces(
                $customerEmbedding,
                $photoEmbeddings
            );
        }
        
        // 5. Return only matched photos
        return array_values(array_filter(
            $results,
            fn($r) => $r->matchesThreshold
        ));
    }
}
```

## Troubleshooting

### Common Issues

1. **Low Similarity Scores**: Check embedding quality and ensure embeddings are from face-api.js
2. **Dimension Mismatch**: Verify embeddings are exactly 128 dimensions
3. **Slow Performance**: Use chunked processing for large albums (>5000 photos)
4. **Memory Issues**: Reduce chunk size or process in smaller batches
5. **No Matches Found**: Try lowering the threshold or verify customer embedding quality

## Summary

The Face Matching Service provides a robust, efficient, and secure solution for face recognition in the Fotlist platform. Key takeaways:

- **Always decrypt embeddings** before passing to the service
- **Use appropriate thresholds** based on your accuracy requirements (default: 0.6)
- **Handle errors gracefully** with try-catch blocks and recovery methods
- **Monitor performance** for large albums and use chunked processing when needed
- **Never log raw embeddings** - the service handles privacy protection automatically

For complete API documentation, see the PHPDoc comments in the service classes.
